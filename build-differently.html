build.differently

The web doesn't have to be a surveillance machine. This is how we build differently.

Principles

Independence

Zero reliance on commercial platforms or services. We own our distribution.

Minimalism

Functional design with as little code as possible. Every byte has purpose.

Longevity

Built to last decades, not update cycles. Future-proof by design.

Transparency

All processes and decisions are visible. No black boxes.

User Respect

No tracking, no data harvesting, no manipulation. You own your data.

Building Blocks

Core Structure

radtypo.github.io/
├── index.html           # Homepage
├── build.differently    # This document
├── radio.html          # Continuous player
└── songs/              # Individual releases
    ├── bad-fortune.html
    ├── muriel.html
    └── whats-not-to-like.html

Release Format

All releases follow: v1.0.0-song-name

releases/v1.0.0-song-name/
├── song-name.mp3           # Main audio
├── song-name-cover-800.jpg # Standard art
├── song-name-cover-400.jpg # Thumbnail
└── song-name-chords.pdf    # Sheet music

Technology

Core Stack

  • Raw HTML/CSS with minimal JavaScript
  • No frameworks, build systems, or dependencies
  • Browser-native audio playback
  • Local storage for preferences
  • Service workers for offline support

Storage

  • GitHub Releases: All media under 100MB
  • Ko-fi: Optional platform for larger files
  • Everything directly downloadable
  • No streaming services or CDNs

Preservation

Data Liberation

  • All data stored locally on your device
  • One-click export of your collection
  • No accounts or authentication
  • Complete offline availability

Future-Proofing

  • Static files preserve functionality
  • No API dependencies
  • Annual static snapshots
  • Content-addressable structure
