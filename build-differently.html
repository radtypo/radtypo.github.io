Guiding Principles

This site operates under strict DIY principles: independence, minimal technology, and respect for our listeners. We view digital technology as a tool for connection, not extraction. We reject the surveillance-based model of the modern web and instead build with the following values:

  1. Independence: Zero reliance on commercial platforms or services
  2. Minimalism: Functional design with as little code as possible
  3. Longevity: Built to last decades, not update cycles
  4. Transparency: All processes and decisions are visible
  5. User Respect: No tracking, no data harvesting, no manipulation

Technical Implementation

Core Architecture

  • Raw HTML/CSS with minimal JavaScript
  • No build systems, frameworks, or dependencies
  • Static site deployment via GitHub Pages
  • Manual deployments with version control
  • Browser-native audio playback (no streaming services)

File Storage & Distribution

  • GitHub Releases: Primary storage for all media files under 100MB
    • Repository: radtypo/radtypo.github.io
    • Consistent release tag format: v1.0.0-song-name
    • Direct download URLs from GitHub's CDN
  • Ko-fi: Optional platform for premium content (stems, sessions)
    • Used only for larger files that exceed GitHub limits
    • Transparent pricing with 100% to artists

Browser Technologies

  • localStorage for preferences and play data
  • Service Workers for offline functionality
  • Native Web Audio API for the player
  • CSS Variables for theming without JavaScript dependencies
  • preload and srcset for responsive resource loading

Ethical Analytics

  • All interaction data stored locally on user's device
  • Play counts and favorites never leave your browser
  • Optional, transparent stats sharing controlled by you
  • Aggregated, anonymous listening data displayed publicly
  • Zero third-party analytics or tracking scripts

Listener Connection

  • Minimalist newsletter signup (locally stored addresses)
  • Manual email distribution (BCC for privacy)
  • "Library card" model for personal collection building
  • Periodic setlist voting for upcoming shows
  • All communication explicitly opt-in with clear purpose

Offline Support

Our Service Worker enables:

  • Offline access to viewed songs
  • Cached catalog data
  • Audio persistence between sessions
  • Background audio playback

Image Delivery

  • Progressive loading for artwork
  • Responsive image sizing based on display
  • Placeholder system during initial load
  • Local caching for faster repeat views

Environmental Considerations

The website is designed for minimal environmental impact:

  • Low-bandwidth content delivery
  • Efficient caching to reduce redundant transfers
  • Text-focused design reduces power consumption
  • Dark mode as default to reduce screen energy usage
  • No auto-playing media or unnecessary animations

Accessibility Commitment

Accessibility is core, not supplemental:

  • Semantic HTML throughout
  • Keyboard navigation for all functions
  • Screen reader compatibility
  • Sufficient contrast in all color modes
  • Text resizing without breaking layouts
  • No reliance on color for critical information

Data Liberation

Your data remains yours:

  • One-click export of your favorites and play history
  • Portable formats (JSON) for your listening data
  • No accounts or authentication requirements
  • All site content available for offline archiving

Preservation Strategy

The site is designed for archival permanence:

  • Static files preserve functionality indefinitely
  • No API dependencies that could disappear
  • Open documentation of all technical decisions
  • Low-tech backup system (annual snapshots)
  • Content addressable structure for future migration
